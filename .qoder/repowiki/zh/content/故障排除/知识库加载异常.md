## 问题排查流程

当知识库初始化失败或内容无法检索时，应按照以下步骤进行系统性排查：

1. **检查服务启动日志**：查看`KnowledgeBaseService.initialize()`方法的日志输出，确认是否出现"知识库服务初始化失败"等错误信息
2. **验证知识库根目录**：确认`knowledge-base`目录是否存在且路径正确
3. **检查子目录结构**：验证`operation-procedures`和`device-apis`子目录是否存在
4. **审查Markdown文件**：检查`.md`文件是否存在、可读且符合元数据格式要求
5. **验证文件编码**：确保所有文件使用UTF-8编码
6. **检查文件权限**：确认应用程序有读取知识库文件的权限
7. **测试文件可读性**：使用调试脚本验证单个文件的可读性

## 语法解析错误处理

`KnowledgeBaseService`对Markdown文件的语法解析采用分层错误处理机制：

### 运维处置文档解析
```mermaid
flowchart TD
StartParse["开始解析运维处置文档"] --> ExtractTitle["从第一行#标题提取title"]
ExtractTitle --> FindMetadata["查找元数据注释<!-- metadata {} --> "]
FindMetadata --> ParseMetadata{"能否解析JSON?"}
ParseMetadata --> |是| UseMetadata["使用解析出的category、keywords、description"]
ParseMetadata --> |否| WarnLog["记录警告日志并继续"]
UseMetadata --> ExtractContent["提取文档内容"]
WarnLog --> ExtractContent
ExtractContent --> AutoExtractKeywords["自动提取关键词"]
AutoExtractKeywords --> CreateEntry["创建KnowledgeEntry实例"]
CreateEntry --> ValidateEntry["验证条目有效性"]
ValidateEntry --> |有效| AddToKnowledgeBase["添加到知识库"]
ValidateEntry --> |无效| ErrorLog["记录错误日志并跳过"]
```


## 缓存加载顺序

知识库服务的缓存加载遵循严格的顺序和依赖关系：

```mermaid
sequenceDiagram
participant App as app.js
participant KBS as KnowledgeBaseService
participant KE as KnowledgeEntry
App->>KBS : initialize()
KBS->>KBS : set knowledgeBasePath
KBS->>KBS : loadKnowledgeBase()
KBS->>KBS : loadOperationProcedures()
KBS->>KBS : readdirSync(operation-procedures)
loop 每个.md文件
KBS->>KBS : readFileSync(filePath, 'utf8')
KBS->>KBS : parseOperationProcedure(content, file)
KBS->>KE : new KnowledgeEntry()
KBS->>KE : validate()
alt 验证成功
KBS->>KBS : addKnowledgeEntry(entry)
KBS->>KBS : 更新categories和keywords索引
else 验证失败
KBS->>Logger : error('知识条目验证失败')
end
end
KBS->>KBS : loadDeviceAPIs()
KBS->>KBS : readdirSync(device-apis)
loop 每个.md/.apib文件
KBS->>KBS : readFileSync(filePath, 'utf8')
KBS->>KBS : parseDeviceAPI(content, file)
KBS->>KBS : addKnowledgeEntry(entry) for each
end
KBS-->>App : 初始化成功
```

## 文件结构完整性检查

- `knowledge-base/` (根目录)
