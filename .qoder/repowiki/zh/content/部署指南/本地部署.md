## 简介
本指南提供在单机环境中部署AutoOperation系统的详细步骤。涵盖前后端服务的独立启动、环境配置、进程守护、反向代理设置以及生产环境必要的安全加固措施，确保系统稳定、安全地运行。

## 项目结构
AutoOperation系统采用前后端分离架构，主要目录包括：
- `backend/`：Node.js后端服务，基于Express框架
- `frontend/`：React前端应用，使用Vite构建
- `config/`：环境配置文件（开发、生产等）
- `configs/`：核心应用和LLM模型配置
- `knowledge-base/`：运维知识库文档

## 后端服务启动
### 手动启动
进入后端目录并启动服务：
```bash
cd backend
npm start
```
默认监听端口为3000，可通过`PORT`环境变量修改。

### 健康检查
服务启动后，可通过以下地址检查状态：
- 健康检查：`http://localhost:3000/health`
- 系统状态：`http://localhost:3000/status`

## 前端服务启动
### 开发模式启动
进入前端目录并启动开发服务器：
```bash
cd frontend
npm run dev
```
默认在端口5173上运行，并通过Vite代理将`/api`请求转发至后端`http://localhost:3000`。

### 生产模式构建与启动
构建静态资源：
```bash
npm run build
```
构建产物位于`dist/`目录，可由任何Web服务器托管。

## PM2进程管理
建议使用PM2管理Node.js后端服务，实现进程守护和自动重启。

### 安装PM2
```bash
npm install -g pm2
```

### 启动后端服务
```bash
cd backend
pm2 start src/app.js --name "autooperation-backend" --watch
```

### 常用PM2命令
```bash
pm2 list                    # 查看进程列表
pm2 logs autooperation-backend  # 查看实时日志
pm2 restart autooperation-backend # 重启服务
pm2 delete autooperation-backend  # 删除进程
pm2 startup                 # 设置开机自启
```

## Nginx反向代理配置
使用Nginx实现前后端同域访问，并配置缓存与压缩。

### 配置示例
```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端静态资源
    location / {
        root /path/to/AutoOperation/frontend/dist;
        try_files $uri $uri/ /index.html;
        expires 1h;           # 静态资源缓存1小时
        add_header Cache-Control "public, immutable";
    }

    # API反向代理
    location /api/ {
        proxy_pass http://localhost:3000/;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }

    # Gzip压缩
    gzip on;
    gzip_vary on;
    gzip_min_length 1024;
    gzip_types text/plain text/css text/xml application/javascript application/json application/xml+rss;
}
```

### 重载Nginx配置
```bash
sudo nginx -t && sudo nginx -s reload
```

## 生产环境安全配置
部署到生产环境时，必须修改以下配置项：

### 1. 关闭调试模式
确保`config/production.json`中相关配置已关闭：
```json
{
  "logging": {
    "level": "warn", // 不使用'info'或'debug'
    "console": { "enabled": false } // 关闭控制台日志
  }
}
```

### 2. 设置强密钥与认证
- **大模型API密钥**：在`config/production.json`中设置`LLM_API_KEY`，避免为空。
- **会话存储**：考虑使用更安全的存储方式（如Redis），而非默认的文件存储。

### 3. 数据库连接信息
若未来集成数据库，需在配置文件中设置正确的连接字符串、用户名和密码。

### 4. CORS策略
严格限制`CORS_ORIGINS`，仅允许受信任的前端域名访问，避免使用通配符。

### 5. 请求限制
调整`RATE_LIMIT_GENERAL`和`RATE_LIMIT_API`以防止滥用。

## 验证与测试
完成部署后，执行以下验证：

1.  **访问前端**：浏览器打开`http://your-domain.com`，确认页面正常加载。
2.  **检查API连通性**：在前端界面发起一个操作，观察网络请求是否成功返回数据。
3.  **查看健康检查**：直接访问`http://localhost:3000/health`，确认所有服务状态为`ok`。
4.  **检查日志**：通过`pm2 logs`或Nginx日志，排查是否有错误信息。
