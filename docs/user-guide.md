## 系统概述

智能运维助手是一个基于AI的智能化运维处置系统，帮助运维人员快速诊断和解决各种系统问题。系统采用人机协作的方式，通过大语言模型分析问题现象，生成结构化的处置方案，并支持自动化执行部分处置步骤。

### 核心功能

- **智能问题分析**: 基于大模型和知识库分析问题现象
- **渐进式处置**: 按步骤引导完成问题处置
- **自动化执行**: 支持API工具的自动执行
- **反馈循环**: 根据用户反馈动态调整处置方案
- **知识积累**: 基于处置经验不断优化知识库

### 系统架构

```
用户界面 → 处置引擎 → 大模型服务
    ↓         ↓         ↓
  会话管理   知识库   工具执行
```

## 快速开始

### 1. 访问系统

打开浏览器，访问：`http://localhost:5173`

### 2. 创建首个处置会话

1. 在首页选择问题分类
2. 详细描述问题现象
3. 点击"开始分析"按钮
4. 系统将自动生成处置方案

### 3. 执行处置步骤

1. 查看生成的处置步骤列表
2. 按顺序执行每个步骤
3. 根据步骤类型选择执行方式：
   - **自动执行**: 点击"自动执行"按钮
   - **手动执行**: 按指导操作后输入结果

### 4. 完成处置

1. 所有步骤执行完成后
2. 系统提示处置完成
3. 可以查看处置总结和建议

## 功能详解

### 问题分类

系统支持以下问题分类：

| 分类 | 描述 | 示例 |
|------|------|------|
| 性能问题 | CPU、内存、磁盘等性能相关 | CPU使用率过高、内存不足 |
| 网络问题 | 网络连接、延迟、丢包等 | 网络超时、连接异常 |
| 服务问题 | 应用服务、数据库等 | 服务无响应、数据库连接失败 |
| 安全问题 | 安全漏洞、入侵检测等 | 异常登录、安全告警 |
| 存储问题 | 磁盘空间、备份恢复等 | 磁盘满、备份失败 |
| 其他问题 | 其他运维相关问题 | 配置错误、环境问题 |

### 步骤类型

#### 手动步骤 (Manual)

需要用户手动执行的操作步骤：

- **特点**: 需要人工判断和操作
- **执行方式**: 按照指导完成操作，然后输入执行结果
- **适用场景**: 需要经验判断、涉及业务逻辑的操作

**示例**:
```
步骤：检查应用日志中的错误信息
指导：
1. 登录服务器
2. 查看 /var/log/app/error.log
3. 找出最近的错误记录
4. 分析错误原因

请输入您发现的错误信息：
[用户输入框]
```

#### 自动步骤 (Auto)

系统可以自动执行的API调用：

- **特点**: 有对应的工具API支持
- **执行方式**: 点击"自动执行"按钮
- **适用场景**: 标准化的API操作、数据查询

**示例**:
```
步骤：获取服务器系统信息
工具：服务器监控API
参数：{ "detail": true }

[自动执行] 按钮
```

#### 分支步骤 (Branch)

根据前一步骤结果决定后续路径：

- **特点**: 有多个可选的后续步骤
- **执行方式**: 系统根据结果自动选择分支
- **适用场景**: 条件判断、多种处置方案

### 用户反馈机制

当遇到问题或需要调整方案时，可以提供反馈：

1. **点击"反馈问题"按钮**
2. **描述遇到的具体问题**
3. **系统重新分析并调整方案**
4. **生成新的处置步骤**

**反馈示例**:
```
原步骤：重启nginx服务
反馈：没有sudo权限，无法重启服务

系统调整：
→ 新步骤：申请管理员权限
→ 新步骤：使用sudo重启nginx服务
```

### 会话管理

#### 会话状态

- **处理中** (Processing): 正在执行处置步骤
- **已完成** (Completed): 所有步骤执行完成
- **已中止** (Aborted): 用户主动中止或系统异常

#### 会话操作

- **查看**: 查看会话详情和步骤执行情况
- **继续**: 继续执行未完成的会话
- **导出**: 导出会话记录（开发中）
- **删除**: 删除不需要的会话记录

### 历史记录

在"历史记录"页面可以：

- **查看所有会话**: 分页浏览历史会话
- **搜索会话**: 按关键词搜索
- **筛选会话**: 按状态、分类筛选
- **管理会话**: 查看、删除会话

**搜索功能**:
- 支持问题描述内容搜索
- 支持会话ID精确查找
- 实时搜索，输入即显示结果

**筛选选项**:
- 状态筛选：所有/处理中/已完成/已中止
- 分类筛选：按问题分类过滤
- 组合筛选：支持多条件组合

## 使用场景

### 场景1：服务器性能问题

**问题现象**: 用户反馈网站访问缓慢

**处置流程**:
1. 选择"性能问题"分类
2. 描述："用户反馈网站访问缓慢，页面加载时间超过10秒"
3. 系统生成处置步骤：
   - 检查服务器CPU使用率 (自动)
   - 检查内存使用情况 (自动)
   - 查看网络连接状态 (自动)
   - 分析应用日志 (手动)
   - 根据分析结果制定优化方案 (手动)

### 场景2：数据库连接异常

**问题现象**: 应用无法连接数据库

**处置流程**:
1. 选择"服务问题"分类
2. 描述："应用日志显示数据库连接超时，用户无法登录"
3. 系统生成处置步骤：
   - 检查数据库服务状态 (自动)
   - 测试数据库连接 (自动)
   - 查看数据库连接数 (自动)
   - 检查网络连通性 (手动)
   - 重启相关服务 (手动确认后自动执行)

### 场景3：安全告警处理

**问题现象**: 监控系统发现异常登录

**处置流程**:
1. 选择"安全问题"分类
2. 描述："监控发现来自异常IP的登录尝试，次数频繁"
3. 系统生成处置步骤：
   - 查看登录日志详情 (自动)
   - 分析异常IP来源 (手动)
   - 检查账户安全状态 (自动)
   - 实施安全防护措施 (手动)

## 最佳实践

### 问题描述技巧

**详细描述问题现象**:
- ✅ 好的描述："服务器CPU使用率持续2小时保持在95%以上，主要是java进程占用，用户反馈系统响应缓慢"
- ❌ 差的描述："服务器有问题"

**包含关键信息**:
- 问题发生时间
- 具体表现症状
- 影响范围
- 已尝试的操作

**使用专业术语**:
- 使用准确的技术名词
- 包含错误码或错误信息
- 提及相关系统和服务

### 步骤执行建议

**执行前准备**:
- 确认操作权限和风险
- 备份重要数据和配置
- 通知相关人员

**执行过程中**:
- 严格按照步骤顺序执行
- 仔细记录操作结果
- 遇到问题及时反馈

**结果反馈**:
- 详细描述操作结果
- 包含相关输出信息
- 说明是否达到预期效果

### 会话管理建议

**命名规范**:
- 使用有意义的问题描述
- 包含时间信息
- 标注紧急程度

**定期清理**:
- 删除无用的测试会话
- 导出重要的处置记录
- 保留典型案例作为参考

## 常见问题

### Q: 如何提高问题分析的准确性？

**A**: 
1. 提供详细的问题描述
2. 包含具体的错误信息
3. 说明问题的影响范围
4. 提及相关的系统环境

### Q: 自动执行步骤失败怎么办？

**A**:
1. 检查网络连接是否正常
2. 确认API权限和配置
3. 查看详细的错误信息
4. 可以改为手动执行该步骤

### Q: 如何处理复杂的多系统问题？

**A**:
1. 将复杂问题拆分为多个子问题
2. 分别为每个子问题创建会话
3. 按优先级和依赖关系执行
4. 注意系统间的影响关系

### Q: 处置步骤太多如何管理？

**A**:
1. 关注当前步骤，逐步执行
2. 利用进度条跟踪整体进度
3. 必要时可以跳过非关键步骤
4. 通过反馈调整处置方案

### Q: 如何积累和共享处置经验？

**A**:
1. 详细记录处置过程和结果
2. 对有效的处置方案给予好评
3. 分享典型案例给团队成员
4. 参与知识库的更新和维护

### Q: 系统响应慢怎么办？

**A**:
1. 检查网络连接状态
2. 确认后端服务是否正常
3. 刷新页面重新尝试
4. 查看浏览器控制台错误信息

## 进阶功能

### 自定义工具集成

系统支持集成自定义的工具API：

1. 按照API Blueprint格式编写工具文档
2. 放置在 `knowledge-base/device-apis/` 目录
3. 重启系统以加载新工具
4. 在处置过程中会自动识别可用工具

### 知识库扩展

可以扩展系统的知识库：

1. 编写Markdown格式的处置文档
2. 包含metadata信息和关键词
3. 放置在 `knowledge-base/operation-procedures/` 目录
4. 系统会自动索引和检索

### 系统设置

在设置页面可以配置：

- **主题设置**: 切换明暗主题、自定义颜色
- **通知设置**: 开启/关闭各种通知
- **语言设置**: 选择界面语言
- **自动保存**: 配置自动保存间隔
